# Performance Pass Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Reclaim frame time at 4000-entity scale by eliminating capacity-bound loops, per-frame allocations, and avoidable trig in hot paths.

**Architecture:** All sim/render systems pivot to packed alive-index iteration. `Entities` already has `aliveIds[0..count)`; we add the same pattern to `Puffs` and `Particles` so every per-frame loop is O(alive) instead of O(capacity). Then we eliminate per-frame allocations (matrices, Maps, closures) and replace `Math.hypot`/`Math.pow` in hot loops with squared-distance comparisons or precomputed values.

**Tech Stack:** TypeScript, vitest, WebGL2, gl-matrix. No new deps.

**Test discipline:** This is a pure perf refactor — behavior must not change. The verification step for every task is `npm test -- --run` showing the same `325 passed, 1 failed` baseline (the failure is pre-existing in `src/lab/wind.test.ts` and unrelated). Visual verification via `npm run dev` after each phase.

**Commit policy:** Don't commit per task. Commit at the end of each phase, after the user reviews.

- [ ] **Step 4: Add a freePuff helper that swap-pops**

Add to the module:
```typescript
/** Frees slot `i`; safe to call from inside a packed-list iteration if you
 *  iterate by index and decrement on free (see updatePuffs). */
export function freePuff(p: Puffs, i: number): void {
  if (p.alive[i] === 0) return;
  p.alive[i] = 0;
  const idx = p.aliveIdx[i]!;
  const last = p.count - 1;
  if (idx !== last) {
    const lastId = p.aliveIds[last]!;
    p.aliveIds[idx] = lastId;
    p.aliveIdx[lastId] = idx;
  }
  p.aliveIdx[i] = -1;
  p.count--;
}
```

- [ ] **Step 5: Update updatePuffs to use packed iteration + swap-pop on death**

Replace `updatePuffs` body. Iterate the packed list by index `n`; when a puff dies, call `freePuff` and **decrement `n`** so the swapped-in slot gets visited:

```typescript
export function updatePuffs(p: Puffs, dt: number): void {
  for (let n = 0; n < p.count; n++) {
    const i = p.aliveIds[n]!;
    const sm = p.sizeMax[i]!;
    const sizeFrac = sm > 0 ? p.size[i]! / sm : 0;
    const decayMul = 1 - (1 - p.decayMul[i]!) * sizeFrac;
    p.life[i] -= dt * decayMul;
    if (p.life[i]! <= 0) {
      freePuff(p, i);
      n--;
      continue;
    }
    const sizeDamp = 1 - p.inertiaWeight[i]! * Math.pow(sizeFrac, p.inertiaExp[i]!);
    const tickMul = p.drag[i]! * sizeDamp;
    p.velX[i] = p.velX[i]! * tickMul;
    p.velY[i] = p.velY[i]! * tickMul;
    p.velY[i] = p.velY[i]! + p.buoyancy[i]! * dt;
    const grown = p.size[i]! + p.edgeGrowth[i]! * dt;
    p.size[i] = grown > sm ? sm : grown;
    p.posX[i] = p.posX[i]! + p.velX[i]! * dt;
    p.posY[i] = p.posY[i]! + p.velY[i]! * dt;
  }
}
```

**Important caveat:** when we free `eaten`, the swap may move a different puff into slot `n`. Two cases:
- If `eaten === i` (we freed the current puff): the swap put a previously-unvisited puff at index `n`, so we want to revisit. Decrement `n`.
- If `eaten !== i` (we freed `bestIdx`): the swap may have moved a puff from a higher index down — that puff has already been processed this frame, but if it lands at `n` (because `aliveIdx[bestIdx] === count - 1`), we'd skip the current `i`. Safer: also decrement `n` in this case to revisit `i`'s slot, but we just processed `i` so we don't want to. Net: only decrement when `eaten === i`.

After `freePuff(p, eaten);`, append:
```typescript
if (eaten === i) n--;
```
