# Gun Crew Phase 2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add per-role tools and a sequential animated reload drill (sponger → loader → rammer → gunner) to the gun crew system shipped in Phase 1.

**Architecture:** Replace the single `gun-crew` `UnitKind` with four role-specific kinds (one per drill role). Each gets its own kit JSON, idle pose with the role's tool baked in, and a 4-frame `reloading` pose with mid-action body+tool composites. `tickCrew` divides the gun's reload cycle into four equal phase windows and drives each crewman's pose state from `parent.reloadT / parent.reloadInitialT`. A new SoA field `reloadInitialT` stores the randomized initial value so windows can be computed correctly.

**Tech Stack:** TypeScript, Vitest, WebGL2 pose-atlas pipeline, procedural pixel-art generation via `pngjs` (matching the existing `draw-cuirassier-components.mjs` pattern).

**Spec:** [`docs/superpowers/specs/2026-04-29-gun-crew-phase-2-design.md`](../specs/2026-04-29-gun-crew-phase-2-design.md)

**Phase 1 plan (for reference):** [`docs/superpowers/plans/2026-04-29-gun-crew-sprites.md`](./2026-04-29-gun-crew-sprites.md)

export function spawnCrewForGun(entities: Entities, gunId: number): number[] {
  const team = entities.team[gunId]!;
  const gunX = entities.posX[gunId]!;
  const gunY = entities.posY[gunId]!;
  const gunFacing = entities.facing[gunId]!;

  const ids: number[] = [];
  for (const role of CREW_ROLES) {
    const kindId = ROLE_KIND_ID[role];
    const kind = getUnitKind(kindId);
    const kindIdx = getUnitKindIndex(kindId);

    const id = allocEntity(entities);
    if (id === -1) {
      console.warn('[crew] entity allocation exhausted; skipping crew spawn');
      break;
    }
    const pose = computeCrewWorldPose(gunX, gunY, gunFacing, role);
    entities.posX[id] = pose.x;
    entities.posY[id] = pose.y;
    entities.restPosX[id] = pose.x;
    entities.restPosY[id] = pose.y;
    entities.facing[id] = pose.facing;
    entities.restFacing[id] = pose.facing;
    const theta = pose.facing * (Math.PI / 4);
    entities.facingIntentX[id] = Math.cos(theta);
    entities.facingIntentY[id] = Math.sin(theta);
    entities.kindId[id] = kindIdx;
    entities.team[id] = team;
    entities.hp[id] = kind.baseStats.hp;
    entities.bodyRadius[id] = kind.baseStats.bodyRadius;
    entities.massKg[id] = kind.baseStats.massKg;
    entities.morale[id] = kind.baseStats.morale;
    entities.state[id] = EntityState.Idle;
    entities.parentGunId[id] = gunId;
    entities.crewRole[id] = role;
    ids.push(id);
  }
  return ids;
}

- [ ] **Step 3: Add pose-driving logic to `tickCrew`**

In `src/sim/crew.ts`:

Add a role-windows table near `ROLE_SPECS`:

```ts
const ROLE_WINDOWS: Record<CrewRole, { start: number; end: number }> = {
  [CrewRole.Sponger]: { start: 0.00, end: 0.25 },
  [CrewRole.Loader]:  { start: 0.25, end: 0.50 },
  [CrewRole.Rammer]:  { start: 0.50, end: 0.75 },
  [CrewRole.Gunner]:  { start: 0.75, end: 1.00 },
};

// fps for the reloading pose, hard-coded to match POSE_CONFIG[Pose.reloading].
// Imported separately so this module doesn't depend on the renderer config.
const RELOAD_POSE_FPS = 6;
const RELOAD_FRAMES = 4;
```

At the bottom of `tickCrew`, add a third pass that runs after the position update:

```ts
  // Pass 3: pose driving. Crew pose is a pure function of parent gun state.
  // Outside Reloading -> idle. Inside Reloading -> active role plays
  // reloading pose with poseT clamped to its phase window; inactive roles
  // play idle.
  for (let id = 0; id < entities.capacity; id++) {
    if (entities.alive[id] !== 1) continue;
    if (!crewKindIdxSet.has(entities.kindId[id]!)) continue;
    const parent = entities.parentGunId[id]!;
    if (parent < 0) continue;

    const parentState = entities.state[parent]!;
    if (parentState !== 4 /* EntityState.Reloading */) {
      entities.pose[id] = 0; // Pose.idle
      entities.poseT[id] = 0;
      continue;
    }

    const initialT = entities.reloadInitialT[parent]!;
    if (initialT <= 0) {
      // Defensive: no valid reload duration recorded.
      entities.pose[id] = 0;
      entities.poseT[id] = 0;
      continue;
    }
    const remainingT = entities.reloadT[parent]!;
    const fullProgress = Math.max(0, Math.min(1, 1 - remainingT / initialT));

    const role = entities.crewRole[id]! as CrewRole;
    const win = ROLE_WINDOWS[role];
    if (fullProgress < win.start || fullProgress >= win.end) {
      entities.pose[id] = 0;
      entities.poseT[id] = 0;
      continue;
    }
    const roleProgress = (fullProgress - win.start) / (win.end - win.start);
    // Map [0,1) across the 4 frames at the configured fps. resolveFrame uses
    // floor(poseT * fps) clamped to frames-1 for oneshot poses.
    entities.pose[id] = 5; // Pose.reloading
    entities.poseT[id] = roleProgress * RELOAD_FRAMES / RELOAD_POSE_FPS;
  }
```

`Pose.idle = 0` and `Pose.reloading = 5` per `pose-config.ts`. Hard-coding the numeric values avoids importing the renderer pose-config into the sim module (the same convention is used elsewhere — see `state-system.ts`'s pose mapping). If you prefer to import `Pose` from `'../render/poses/pose-config'` for clarity, that's also acceptable.
