# Cavalry & Cannon Basic Sprite Assets

## Goal

Bring cuirassier (cavalry) and cannon-12 (artillery) up to visual parity with line-infantry: each unit kind has an 8-facing sprite atlas with team-tinted primary/secondary markers, generated by both a runtime TypeScript module (used by the GL sprite-pass) and a Node draw script (used to produce static PNG previews under `public/sprites/`).

Today both unit kinds render as flat tinted rectangles because they have no `spriteCell` and the sprite-pass falls back to the white tint cell of the soldier atlas.

## Non-goals (v1)

- No charge / fire / recoil animation frames â€” only the eight static facings.
- No artillery crew sprites â€” the cannon is rendered without gunners. Crew is a follow-up task.
- No new team palette work â€” the existing `TEAM_COLORS` (British, French) continues to drive primary/secondary tinting.
- No collision / sim changes â€” `placeholderSize`, `bodyRadius`, `barrelOffset`, etc. on the unit kinds stay untouched.

## Architecture

### Single combined atlas, per-kind row groups

The current sprite-pass binds one texture and uses one `POSE_CELLS` table shared across all kinds. The pass will be extended so the combined atlas is the vertical concatenation of per-kind 3Ã—3 cell grids â€” one block per kind â€” and so the per-kind pose-cell table is selected at draw time by `kindId`.

Atlas dimensions:

| Kind         | Cell W Ã— H | Region W Ã— H | V offset (rows) |
|--------------|-----------:|-------------:|----------------:|
| Soldier      | 11 Ã— 18    | 33 Ã— 54      | 0               |
| Cuirassier   | 15 Ã— 20    | 45 Ã— 60      | 54              |
| Cannon-12    | 17 Ã— 14    | 51 Ã— 42      | 114             |
| **Combined** |     â€”      | **51 Ã— 156** | â€”               |

Each kind's region is a 3-cells-wide Ã— 3-cells-tall grid laid out exactly like the soldier atlas:

```
[NW back-3/4 mirrored] [WHITE TINT]   [NE back-3/4]
[W side mirrored]      [S front]      [E side]
[SW front-3/4 mirrored][N back]       [SE front-3/4]
```

The sheet width is `max(per-kind cell-W) Ã— 3 = 51`. Kind regions narrower than 51 are padded on the right with transparent pixels (their UVs only sample the left portion). The sheet height is the sum of per-kind region heights.

### Why this layout

- One texture bind, one draw call per frame â€” preserves the current GPU shape.
- The marker-replacement shader (magenta â†’ primary, cyan â†’ secondary) is unchanged; tints continue to work per-instance.
- New unit kinds can be appended to the bottom of the atlas without affecting existing UV math.
- Padding waste is small (combined atlas is 51Ã—156 = ~8 KB of RGBA at native res); not worth a packing algorithm.

### Module shape

```
src/render/
  sprite-atlas.ts                 (new) - combined atlas builder + per-kind metadata
  cuirassier-sprite.ts            (new) - cuirassier RGBA generator + pose grids
  cannon-12-sprite.ts             (new) - cannon RGBA generator + pose grids
  british-soldier-sprite.ts       (existing) - unchanged structure, exports kept
  passes/sprite-pass.ts           (modified) - uses combined atlas + per-kind pose cells
```

`sprite-atlas.ts` exports:

```ts
export interface KindAtlasMeta {
  cellW: number;
  cellH: number;
  poseCells: readonly { col: number; row: number }[]; // 8 entries, N..NW clockwise
  frontCell: { col: number; row: number };
  tintCell: { col: number; row: number };
  /** Pixel offset of this kind's region within the combined atlas. */
  region: { x: number; y: number; w: number; h: number };
}

export const KIND_ATLAS: Record<string, KindAtlasMeta>;
export const COMBINED_SHEET_W: number;
export const COMBINED_SHEET_H: number;
export function generateCombinedAtlas(opts?: { resolvePrimary?: RGB; resolveSecondary?: RGB }): Uint8Array;
```

Internally it calls each kind's RGBA generator (`generateBritishSoldierSheet`, `generateCuirassierSheet`, `generateCannonSheet`) and blits the result into the combined buffer at the right offset.

### Sprite-pass changes

`createSpritePass` swaps the soldier-only atlas for `generateCombinedAtlas()` and replaces the single `POSE_CELLS` import with a `kindId â†’ KindAtlasMeta` lookup. Per draw:

1. Read the entity's `kindId` and `facing`.
2. Look up `meta = KIND_ATLAS[kindString]`.
3. Pick the cell: `meta.poseCells[facing - 1]` if `facing âˆˆ 1..8`, else `meta.frontCell`.
4. Compute the UV rect in the **combined atlas** coordinates: cell column/row are local to the kind's region, then translated by `meta.region.{x,y}` and divided by `COMBINED_SHEET_{W,H}`.

Per-instance scratch arrays already carry `cellW/cellH` implicitly via `placeholderSize` (world-space) and the UV rect's pixel size is encoded in `scratchUv`. No new GPU attributes needed.

### Data wiring

`UnitKind.spriteCell` keeps its current meaning (the front cell, used as a non-facing fallback). For cuirassier and cannon-12 it is set to that kind's front cell â€” this lets non-facing-aware code paths continue to work, and keeps the type unchanged.

Atlas metadata (`cellW`, `cellH`, `poseCells`, `region`) is held centrally in `sprite-atlas.ts` rather than on each `UnitKind`, because it is a render-only concern and the sim should not depend on it.

## Sprite specs

### Cuirassier â€” 15 Ã— 20 cells

- Horse profile: chest, four legs in a running stance, tail. Side cell shows a clear gallop silhouette.
- Rider in cuirass + helmet sits on horse back; sabre held in viewer-right hand on side and front cells; on back cells the sabre flips to the rider's left for visual mirror consistency.
- Color markers:
  - **Primary (P)**: rider's coat sleeves and breast over the cuirass.
  - **Secondary (S)**: cuirass plate and helmet plume / horsehair crest.
- Horse coat: a fixed mid-brown (`[110, 75, 45]`) â€” not faction-tinted in v1.
- 3/4 (NE/NW/SE/SW) cells reuse the same horse silhouette nudged 1px and shift the rider's helmet plume / sabre to indicate orientation, mirroring the soldier diag conventions.
- World size unchanged: `placeholderSize { w: 1.4, h: 2.4 }`.

### Cannon-12 â€” 17 Ã— 14 cells

- Two-wheeled gun carriage with a long barrel, no crew.
- Side cell: barrel pointing right, both wheels visible behind the trail.
- Front / back: barrel end-on (a small dark circle for the muzzle / breech) with wheels splayed left and right, trail pointing toward / away from camera.
- 3/4 cells: barrel angled and one wheel partially occluding the other.
- Color markers:
  - **Primary (P)**: carriage timber along the trail and cheeks.
  - **Secondary (S)**: wheel rims / iron tyres, barrel furniture (cap-square, breech ring).
- Barrel: gunmetal grey (`[60, 62, 70]`), not tinted. Wood: a fixed warm brown for unfaction-specific surfaces (`[120, 84, 50]`).
- World size unchanged: `placeholderSize { w: 2.2, h: 2.8 }`.

## Risks / open questions

- **Cell-size mismatch**: kinds with different cell sizes share one quad in the sprite-pass. UV rect width/height already vary per-instance via `scratchUv`, so this is supported, but the test must catch off-by-one half-texel insets.
- **World-size vs cell aspect ratio**: cuirassier cell aspect is 15:20 = 0.75; world size is 1.4:2.4 â‰ˆ 0.583. The shader stretches the sprite quad to `placeholderSize` so the on-screen aspect is determined by world size, not the cell. If this looks wrong in practice we adjust cell dimensions in a follow-up â€” reversible.
- **Cannon facing semantics**: the sim treats facing as 1..8 (N..NW). For artillery without crew this might look static / robotic at low rotation rates, but matches infantry semantics. Acceptable for v1.
